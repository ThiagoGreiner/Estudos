# TÉCNICAS DE TESTE

Cada nível de um teste vai possuir técnicas de testes diferentes

### TÉCNICA DE CAIXA BRANCA

Tambem conhecido como teste estrutural, é conhecer o código fonte e como funciona por "baixo dos panos"

* Também conhecido como teste estrutural
* Validar dados, controles, fluxos, chamadas
* Garantir a qualidade da implementação
* Níveis: unidade, integração, regreção

### TÉCNICA DE CAIXA PRETA

Tambem conhecido como teste funcional, é um teste que é realizado sem conhecer o software com atecedência

* Também conhecido como Teste funcional
* Verificar saídas usando vários tipos de entrada
* Teste sem conhecer a estrutura interna do software
* Níveis: integração, sistema, aceitação

### TÉCNICA DE CAIXA CINZA

Este teste mescla os testes de caixa branca e caixa preta

* Mescla técnicas de Caixa Branca e Caixa Preta
* Analisa parte lógica e também funcionalidade
* Exemplo: Ter acesso a documentação do funcionamento do código
* Engenharia Reversa

Dois exemplos de testes de caixa cinza são

* Um desenvolvedor que ao mesmos tempo tempo que esta desenvolvendo o software ele vai realizando testes
* Um testador que mesmo sem conhecer o software, ele tem acesso a documentação do mesmo
